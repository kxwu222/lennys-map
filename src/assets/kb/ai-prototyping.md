## Summary
Colin Matthews walks through the AI prototyping landscape for PMs. Three tool categories: chatbots (Claude, ChatGPT) for simple single-page prototypes, cloud dev environments (Replit, Bolt, v0, Lovable) for multi-page apps with hosting, and local dev assistants (Cursor, GitHub Copilot, Windsurf) for production-grade code. PMs can now turn Figma designs, hand-drawn sketches, or PRDs into working prototypes in minutes with no coding ability.

## Key Insight 1
Tool selection guide: v0 for beautiful designs by default (uses Next.js/Shadcn UI), Bolt for quick prototypes with flexible designs (runs server in-browser), Replit for internal tools or data-driven apps (full-stack with Python support), and Lovable for production apps with integrations (GitHub, Supabase, AI providers). Chatbots work for very simple one-page prototypes like calculators or data visualizations.

## Key Insight 2
Four strategies for solving prototyping problems without coding: Reflection — force the AI to plan before coding ("detail the requirements, do not write code"); Batching — build smallest functional iteration first, then extend; Be specific — describe every change in detail, specify technologies, files, and behaviors; Lost context — use checkpoint/rollback systems when AI rewrites too much. Combine all three to minimize errors.

## Key Insight 3
The post includes six ready-to-use prompt templates: converting Figma designs to prototypes, building from scratch with good UI defaults, creating data visualization dashboards, converting hand-drawn mockups, turning PRDs into interactive prototypes, and building personalized productivity tools. Each template specifies the best tool to use and includes real deployed examples.
