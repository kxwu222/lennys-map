## Summary
Zevi Arnovitz, a PM at Meta with zero technical background, shares how he uses Cursor and Claude Code to build and ship real products. He describes a practical workflow for non-technical people to leverage AI coding tools, covering everything from generating code to reviewing it. The episode makes a compelling case that titles and responsibilities are collapsing as AI democratizes software creation.

## Key Insight 1
Arnovitz uses a dual-tool review strategy: he runs Cursor and Codex side by side, having each AI review the other's code. Since non-technical PMs struggle to catch mistakes in AI-generated code, this cross-checking approach creates a safety net. The /review command in Claude tells the AI to audit its own output, but getting a second AI opinion adds another layer of quality assurance.

## Key Insight 2
The core thesis is that AI coding tools have given non-technical people genuine superpowers. When Sonnet 3.5 launched, Arnovitz realized he could build apps using tools like Bolt and Lovable just by describing what he wanted. The barrier is no longer whether you can code but whether you can clearly articulate what you want built and iterate effectively on AI output.

## Key Insight 3
Arnovitz argues this is the best time to be a junior or non-technical builder, contrary to widespread fear about AI replacing jobs. The people who will thrive are those who learn to use AI tools effectively. His practical advice: start building, treat AI like a remote engineer you are directing, and focus on getting better at prompting and reviewing rather than learning to code from scratch.
